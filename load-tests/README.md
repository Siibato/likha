# Likha Load Tests

k6 load testing suite for the Likha LMS server. TypeScript + pnpm + esbuild.

## Prerequisites

```bash
brew install k6
```

## Setup

```bash
# 1. Install dependencies
pnpm install

# 2. Configure environment
cp .env.example .env
# Edit .env — set BASE_URL to your Pi's IP address

# 3. Generate seed manifest (resets DB, seeds, and exports manifest in one step)
# From the project root:
make db-seed-manifest

# 4. Build all scenarios
pnpm build
```

## Running Tests

```bash
# Increase socket limit (macOS — required before each k6 run)
ulimit -n 65535

# Baseline: verify server is healthy (50 VU, 2 min)
pnpm test:baseline

# Stress: find breaking point (50 → 500 VU ramp)
pnpm test:stress

# Classroom mix: teachers + students concurrent (75 VU)
pnpm test:classroom

# Exam surge: 70 students burst into same assessment simultaneously
pnpm test:exam

# Delta sync: delta vs full sync comparison (100 VU)
pnpm test:delta

# Teacher grading: grading burst + grade compute (5 VU)
pnpm test:grading

# Quick profiler: all endpoints, 50 VU, 1 min — fast outlier detection
pnpm test:all-endpoints-quick

# Run all in sequence
pnpm test:all

# Export results to CSV
export $(grep -v '^#' .env | xargs) && k6 run --out csv=results.csv dist/stress.js
```

## Project Structure

```
src/
├── config/      # env, thresholds, stages — all tunable in one place
├── core/        # api-client, auth-manager, check-helpers, scenario-builder
├── data/        # accounts (76 seeded users), manifest (seeded IDs), fixtures
├── services/    # one file per server module: auth, class, assessment, ...
├── scenarios/   # entry points for k6: baseline, stress, classroom-mix, ...
└── types/       # TypeScript interfaces for API DTOs and scenario types
```

|------|-----------|----------|-------|
| Teacher | `teacher_01` – `teacher_05` | `teacher123` | 5 |
| Student | `student_01` – `student_70` | `student123` | 70 |

## Seed Manifest

`seed-manifest.json` is generated by the server and contains all seeded entity IDs
(classes, assessments, assignments, users). It is gitignored — regenerate it whenever
you reset the database.

```bash
# Reset DB + seed + export manifest in one go (from project root)
make db-seed-manifest
```
